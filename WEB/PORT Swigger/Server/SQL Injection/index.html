How to Stop SQL Injection

The absolute best way to prevent SQL Injection is by using parameterized queries (also known as Prepared Statements). When you use prepared statements, the database treats user inputs strictly as safe text strings, and never compiles them as executable code instructions.

Compare SQL Compilers In Real-Time

Type an injection string below and click test. See how vulnerable dynamic SQL compilers run it, while parameterized statements isolate and neutralize it completely.

Insecure Dynamic Concatenation Java 
String query = "SELECT * FROM products WHERE category = '" + input + "' AND released = 1";
Statement stmt = connection.createStatement();
ResultSet rs = stmt.executeQuery(query);
COMPILED DATABASE QUERY:
Secure Parameterized Statement Java 
PreparedStatement stmt = connection.prepareStatement(
    "SELECT * FROM products WHERE category = ? AND released = 1"
);
stmt.setString(1, input);
ResultSet rs = stmt.executeQuery();
